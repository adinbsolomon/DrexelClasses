Practicum Exam 2

For this exam, you’ll write a full-stack web application to display a list of which trains are next to arrive for a given regional rail stop along the Manyunk/Norristown line, using SEPTA’s Next to Arrive API.

Starter Code

The code in the provided starter/ folder has this directory structure:

starter
├── app
│   ├── public
│   │   ├── client.js
│   │   └── index.html
│   └── server.js
├── env.json

To run your server:

  • cd into starter/ and run npm install to install the axios and Express packages.
  • cd into starter/app and run node server.js to start your server.
  • Visit http://localhost:3000 to see the index.html file.

The API

We’ll use SEPTA’s Next to Arrive API. This API returns a list of trains that are next to arrive at a given origin station, going towards a given destination station. The URL format is:

http://www3.septa.org/hackathon/NextToArrive/Origin Station/Destination Station

A complete list of valid stations for the Manayunk/Norristown line is already provided a) in the dropdown options in index.html and b) in a list of strings in server.js.

For example, if we send a GET request to the URL

http://www3.septa.org/hackathon/NextToArrive/Elm St/Norristown TC

we should see something like this (depending on the time of day and what trains will arrive soon):

[
    {
        "orig_train": "2323",
        "orig_line": "Manayunk/Norristown",
        "orig_departure_time": "4:08PM",
        "orig_arrival_time": "4:14PM",
        "orig_delay": "113 mins",
        "isdirect": "true"
    },
    {
        "orig_train": "2329",
        "orig_line": "Manayunk/Norristown",
        "orig_departure_time": "7:08PM",
        "orig_arrival_time": "7:14PM",
        "orig_delay": "On time",
        "isdirect": "true"
    }
]

For this exam, you don’t need to worry about encoding spaces in URLs. All modern browsers will handle them appropriately.

Your Task

Add server-side code to server.js that does this:

  • Create a request handler for the route GET /next.
  • In this request handler, extract the origin and destination keys from the query string.
  • If the origin or destination values aren’t in the list of stations, respond with status 400 and this exact JSON body: {error: "Invalid origin or destination"}.
  • If the origin and destination stations are equal, respond with status 400 and this exact JSON body: {error: "Origin and destination must be different"}.
  • Otherwise, if the origin and destination stations are valid, using axios, send a GET request to the SEPTA API.
  • You must use the baseUrl variable to construct your request URL or you’ll fail our unit tests.
  • Respond with status 200 and a JSON body containing an object with the key trains where the value is the raw list of trains received from the SEPTA API, e.g.: {trains: [... list of trains from API ...]}.
  • You don’t need to catch errors from the axios request for this exam.

Add client-side code to client.js that does this:

  • When the button with id get-button is clicked (defined in index.html), send an Ajax request to the GET /next route using the Fetch API.
  • This request should have a query string with two parameters: origin and destination.
  • The values of these parameters should be taken from the value of the currently selected option element in the dropdowns with ids origin-select and destination-select, respectively.
    • You may reference this Stack Overflow question to write code to do this.
  • If the response has a status code of 200, you should create a row for each train and append it to the table body with id train-table.
    • The “Train”, “Departure Time”, “Arrival Time”, and “Delay” columns should contain the raw text from the orig_train, orig_departure_time, orig_arrival_time, and orig_delay values, respectively.
    • If a train’s orig_delay is equal to "On time", color the row with a lightgreen background color.
  • Otherwise, if the response has any other status code, you should display the text under the key error from the JSON response body in the div with id message-div.
  • Each request should clear the previous trains and the previous error message, if any.
